# SDKWork Course TODO Completion Plan

**Goal:** Complete all ~113 `todo!()` stubs and TODO comments across the 4-crate sdkwork-course workspace, producing a fully functional course management system.

**Architecture:** Service crate owns business rules and port traits. Repository crate implements those traits via adapters over `CourseSqlxRepositoryPort`. Router crates are thin HTTP adapters calling service traits. Provider ports get no-op stubs.

**Tech Stack:** Rust 1.92.0 (native async traits), SQLx (SQLite/Postgres), serde/serde_json, chrono, uuid. No HTTP framework yet — handlers return `serde_json::Value`.

## Risk Assessment

| Risk | Mitigation |
|------|------------|
| Async trait compilation issues | Rust 1.92.0 supports native async traits; test Phase 1 early |
| Service trait method explosion (~50+ methods) | Split into sub-traits if needed; keep focused |
| Handler parameter extraction without framework | Handlers take explicit params; framework extractors map later |
| Repository adapter type complexity | Use `dyn CourseSqlxRepositoryPort` trait objects |
